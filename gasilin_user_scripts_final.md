### [ Сценарий 1: регистрация пользователя/вход ]
1.	При нажатии кнопки «регистрация» пользователь переходит в активность регистрации
2.	Пользователь вводит имя пользователя
3.	Пользователь вводит логин
4.	Пользователь вводит пароль
5.	Пользователь нажимает на кнопку «зарегистрироваться»
6.	Система проверяет, существуют ли пользователи с таким логином или именем пользователя
7.	В случае наличия пользователя с логином, аналогичным введенному, пользователю будет выведено сообщение что такой логин уже занят
8.	В случае наличия пользователя с именем пользователя, аналогичным введенному, пользователю будет выведено сообщение что такое имя пользователя уже занято
9.	В случае отсутствия совпадений по логину или паролю новый пользователь будет добавлен и пользователь вернется в активность входа
10.	Пользователь вводит логин
11.	Пользователь вводит пароль
12.	Пользователь нажимает кнопку «войти»
13.	Система проверяет наличие пользователя с логином и паролем, аналогичными введенным
14.	В случае отсутствия совпадений пользователь получает сообщение, что неправильно введен логин и/или пароль
15.	В ином случае запускается основное окно приложения, пользователь переходит к выбору дальнейших действий

### [ Сценарий 2: настройка параметров записи на мероприятие]

1.  Пользователь при создании мероприятия выбирает параметры записи
2.  При необходимости пользователь может настроить параметры записи и посмотреть количество записавшихся 
3.  Пользователь переходит в просмотр своего мероприятия
4.  Пользователь переходит в раздел «запись»
5.  Пользователю показано количество записавшихся и информация о записи
6.  Пользователь может изменить параметры записи, настроив их и нажав на кнопку «сохранить изменения»
7.  Вернуть изменения к последней сохраненной версии пользователь может, нажав кнопку «отменить изменения»

### [ Сценарий 3: размещение анкеты мероприятия ]

1.	Пользователь вводит название мероприятия
2.	Пользователь вводит описание мероприятия
3.	  При нажатии кнопки «выбрать место»  будет запущена активность выбора мероприятия. На ней пользователь выбирает на карте место проведения мероприятия. При нажатии на кнопку «готово» адрес и координаты мероприятия будут переданы в активность создания мероприятия
4.	Пользователь может ввести вручную адрес или координаты мероприятия и по ним получить оставшиеся элементы
5.	Пользователь вводит дату проведения мероприятия в формате ДДММГГГГ
6.	Пользователь настраивает параметры запись (наличие записи, ограничения на максимальное количество записей, ограничение на максимальное количество запесей)
7.	Пользователь выбирает, к каким категориям принадлежит мероприятие
8.	Пользователь нажимает на кнопку «готово»
9.	Прервать создание мероприятия можно нажатием на кнопку «отмена»
10.	Система проверяет корректность заполнения полей (формат даты, наличие адреса, имени, описания, координат и категорий мероприятия)
11.	В случае корректного заполнения полей мероприятие будет добавлено и пользователь будет возвращен в основное окно приложения для перехода к выбору дальнейших действий

### [ Сценарий 4: поиск мероприятия по категориям ] 

1.	Пользователь переходит в раздел «категории»
2.	Если пользователь ранее использовал поиск по категориям, то категории с прошлого раза будут установлены
3.	Пользователь настраивает категории
4.	Система сохраняет установленные категории
5.	Пользователь переходит в раздел «поиск»
6.	Мероприятия будут показаны в соответствии с категориями 
7.	Пользователь при введении имени мероприятия в поле поиска переходит в сценарий 5 – поиск мероприятия по названию
8.	При нажатии на мероприятие будет показано окно с базовой информацией о мероприятии 
9.	При нажатии в окне на кнопку «посмотреть» пользователь переходит в активность просмотра мероприятия

### [ Сценарий 5: поиск мероприятия по названию ]

1.	Пользователь переходит в раздел «поиск»
2.	Изначально будут выведены мероприятия, являющиеся результатом сценария 4
3.	Пользователь вводит название мероприятия
4.	Система проверяет наличие мероприятий с таким названием или названием, содержащим введенный текст
5.	Пользователю будут показаны искомые мероприятия
6.	При нажатии кнопки «сброс» пользователь перейдет в сценарий 4
7.	При нажатии на мероприятие будет показано окно с базовой информацией о мероприятии 
8.	При нажатии в окне на кнопку «посмотреть» пользователь переходит в активность просмотра мероприятия

### [ Сценарий 6: поиск мероприятий по удаленности ]

1.	Пользователь переходит в раздел «категории»
2.	Если пользователь ранее не настраивал местоположение, то замеряться расстояние будет от центра Москвы
3.	При нажатии на кнопку «задать местоположение» пользователь перейдет в активность выбора местоположения
4.	После выбора местоположения (если пользователь ранее его уже выбирал, то оно будет загружено), пользователь выставляет на ползунке расстояние
5.	Пользователь переходит в раздел «поиск»
6.	Пользователь нажимает на кнопку «поиск по удаленности»
7.	Система подбирает мероприятия, удаленные на расстояние, меньше или равное чем указанное пользователем
8.	Если пользователь выбрал «поиск по категориям», то подобранные мероприятия будут соответствовать и выбранным категориям
9.	Мероприятия будут показаны на карте в виде маркеров
10.	При нажатии на мероприятие будет показано окно с базовой информацией о мероприятии 
11.	При нажатии в окне на кнопку «посмотреть» пользователь переходит в активность просмотра мероприятия

### [ Сценарий 7: визуализация мероприятий на карте ]

1.	Пользователь переходит в раздел «карта»
2.	Система подбирает мероприятия соответствующие критериям
3.	В случае если пользователь устанавливал критерии поиска, то мероприятия на карте будут показаны в соответствии с критериями 
4.	В случае если пользователь не устанавливал критерии поиска, то система будет показывать на карте мероприятия всех типов
5.	На карте будут визуализированы соответствующие мероприятия 
6.	Мероприятия на карте будут показаны в виде маркеров
7.	При нажатии на маркер мероприятия пользователю будет показана базовая информация о мероприятии
8.	При нажатии в окне с информацией о мероприятии кнопки «посмотреть» пользователь перейдет в активность просмотра мероприятия

### [ Сценарий 8: визуализация мероприятий определенных типов на карте ]

1.	Пользователь переходит в раздел «категории»
2.	Пользователь выставляет категории поиска
3.	Пользователь переходит в раздел «карта»
4.	Система подбирает мероприятия, соответствующие критериям поиска
5.	Подобранные мероприятия отображаются на карте в виде маркеров
6.	При нажатии на маркер мероприятия пользователю будет показана базовая информация о мероприятии 
7.	При нажатии в окне с информацией о мероприятии кнопки «посмотреть» пользователь перейдет в активность просмотра мероприятия

### [ Сценарий 9: публикация отзывов о мероприятии ]

1.	Пользователь в любом из разделов нажимает на мероприятие
2.	Пользователю высвечивается окно с базовой информацией о мероприятии
3.	При нажатии на кнопку «посмотреть» пользователь переходит в активность просмотра мероприятия
4.	Пользователь переходит в раздел «комментарии», в котором отображены комментарии к мероприятию
5.	Пользователь вводит текст отзыва
6.	Пользователь нажимает кнопку «отправить»
7.	Система проверяет, не пустой ли отзыв
8.	В случае отсутствия текста в отзыве пользователю сообщается, что поле текста пустое
9.	В ином случае отзыв добавляется

### [ Сценарий 10: запись на мероприятия ]

1. Пользователь в любом и разделов нажимает на мероприятие
2. Пользователю высвечивается окно с базовой информацией о мероприятии
3. При нажатии на кнопку «посмотреть» пользователь переходит в активность просмотра мероприятия
4. Пользователь переходит в раздел «запись»
5. Если пользователь не является создателем мероприяти, то ему будет предложена возможность зарегистрироваться в случае, если регистрация возможна (открыта регистрация и не превышен лимит на количество записей
6. Если пользователь уже зарегистрирован, то он сможет отменить регистрацию
7. В случае, если пользователь - создатель мероприятия, он переходит в сценарий 2

### [Сценарий 11: просмотр мероприятий, на которые записался пользователь ]

1. Пользователь переходит в раздел «запись»
2. Система подбирает мероприятия, на которые записан пользователь
3. В случае, если пользователь зарегистрировался на мероприятие, но впоследствии запись на него была отключена, то это мероприятие не будет отображено в списке
4. В случае, если пользователь зарегистрировался на мероприятие, но ограничение на максимальное количество записей было уменьшено так, что пользователь уже не входит в количество записанных, то мероприятие не будет отображаться в списке 
5. В случаях 3 и 4 регистрация будет восстановлена, если условия будут соблюдены (наличие записи и/или ограничение на количество записей) 
6. Мероприятия выводтся на экран в виде списка
7. При нажатии на элемент списка появляется окно с основной информацией о мероприятии
8. При нажатии кнопки «посмотреть» пользователь переходит в активность просмотра мероприятия
